MarkDown学习记录

1.0 关于

1.1 关于

从网络上听说markdown的语法简单到两分钟就可以学会，于是乎就开是作死地在这里写第一篇博文的时候用上了。
本人使用SpaceVim写markdown倒是有些违背了一步一步慢慢来的初衷，不过既然是使用编辑器又有何紧要呢。 在看别人的文档的时候看得一个牛逼的故事，一个关于辍学的牛逼故事。详情可以google markdown的作者。 markdown语法十分的简洁好用，相较于word的复杂段落、字体等的设置来说，的确可以称得上是两分钟学会的语法了。 简洁的语法使得写作者更加注重于写作文本的内容，并且同样可以拥有美观的外表。并且像字体的设置对于阅读而言自然是舒服即可，无须种种精细。

2.0 markdown语法

2.1 标题

标题有两种表示方法，私心觉得第二种较为好用。写作时却是第一种显得清晰简洁，预览效果都是一样的。因为第二种所分的级数更多，所以觉得会更加常用。

1、=与-

文字下的'========'可以表示一级标题(无个数限制)
文字下的'--------------'可以表示二级标题(无个数限制)

2、‘#’

n个’#‘在文字前面可以表示n级标题，最多为6级标题

2.2 段落

换行即为段落

如果要段内强制换行则在句末加上两个及以上的空格加回车（引用中换行省略回车）

2.3 区块引用

在段落的第一行使用 > 符号即可，在引用中使用 >> 还可以表示嵌套
>引用
>>引用中引用

2.4 代码区块

既然markdown的发明者是一名极客肯定少不了代码的引用。
代码区块建立只需要在每行加上4个空格 或者两个制表符
需要注意的是在强制换行中无用，如需建立代码区块仍然需要空一行即另起一段。 如下：

import tesnsorflow as tf
import re

2.5 强调

在强调内容的两侧加上*或者_,两侧各加一个表示斜体，两侧各加两个表示斜体 另外因为markdown兼容html语法所以如果不限麻烦还可以使用我们熟知的<>中间加b来加粗 示例：

2.6 列表

使用·、+、或-标记无序列表，标记后至少有一个空格

  • 第一项
  • 第二项
  • 第三项

使用数字加上.进行有序标记，同样标记后至少一个空格

  1. 第一
  2. 第二
  3. 第三

2.8 链接

链接有两种形式，分别为行内式和参考式

1.行内式：

[mygithub](http://www.github.com/kingrongH "mygithub")

效果：

mygithub

后""中的文档为提示文字

2.参考式：

[mygithub]1 1:www.github.com/kingrongH "mygithub"

[注]：第二行不得出现在区块(引用区块或者代码区块)中，否则无用
效果：

mygithub
